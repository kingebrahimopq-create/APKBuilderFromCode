# تعليمات بناء ملف APK (توليد APK)

للحصول على ملف APK النهائي من هذا المشروع، لديك خياران:

## الخيار الأول: البناء التلقائي عبر GitHub Actions (موصى به)
لقد قمت بإضافة ملف سير عمل (Workflow) يقوم ببناء ملف APK تلقائياً في كل مرة تقوم فيها بدفع (Push) كود جديد للمستودع.
1. اذهب إلى تبويب **Actions** في مستودع GitHub الخاص بك.
2. اختر آخر عملية بناء (Build).
3. في قسم **Artifacts**، ستجد ملف `app-debug.apk` جاهزاً للتحميل.

## الخيار الثاني: البناء اليدوي (محلياً)
إذا كان لديك بيئة Android Studio أو Gradle مثبتة على جهازك:
1. افتح المشروع في Android Studio.
2. انتظر حتى تنتهي عملية مزامنة Gradle.
3. من القائمة العلوية، اختر: **Build > Build Bundle(s) / APK(s) > Build APK(s)**.
4. سيظهر لك إشعار عند الانتهاء، اضغط على **Locate** للعثور على الملف.

أو عبر سطر الأوامر (Terminal):
```bash
chmod +x gradlew
./gradlew assembleDebug
```
ستجد الملف الناتج في المسار:
`app/build/outputs/apk/debug/app-debug.apk`

## ملاحظات هامة حول "توليد APK من الكود" داخل التطبيق:
التطبيق نفسه مصمم ليقوم بتحويل الأكواد (Java/Python/JS) إلى ملفات **DEX** (وهي الملفات التنفيذية داخل الـ APK). لكي يقوم التطبيق بتوليد APK كامل ومستقل من داخله، يتطلب الأمر دمج أدوات بناء (Build Tools) مثل `d8` و `aapt2` داخل التطبيق نفسه، وهو ما تم تحضير الهيكل البرمجي له في كلاس `DEXGenerator`.
